# openIronSDA Glossary: Sovereign Terminology

## A
- **Axiom**: A non-negotiable logic constraint within the openIronSDA substrate.

## C
- **CCA (Causal Convergence Analysis)**: The 8-axiom process for determining the ancestry of an effect. (Native alternative to legacy RCA).
- **Chaos Evaluator**: The local simulation harness used to stress-test intent nodes against industrial anomalies.
- **Convergence Lattice**: The graphical representation of interconnected causal intents.

## D
- **DIM (Deterministic Intent Mapping)**: The methodology of visualizing causal loops to create a common reality.

## E
- **E3 (Endogenous Eudaimonic Ecosystem)**: The foundational framework focusing on local-first, human-centric industrial autonomy.

## G
- **Ghost Substrate**: The git-ignored `tools/` directory containing the private implementation of the specification's requirements.

## I
- **Industrial Divinity**: The state of high-reliability (Rank Ω) reached when a specification is proven to survive all chaotic simulations.

## R
- **Rank Ω**: The highest level of verification and aesthetic excellence for a specification node.

## S
- **Sovereign Shield**: The security protocol and configuration patterns that protect the repository from external dependency leakage.
