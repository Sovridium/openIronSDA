# Contributing to openIronSDA

Since **openIronSDA** is an Intent-Driven Development (IDD) project, the contribution process differs from traditional coding repos.

## 🤝 How to Contribute Intent
1.  **Draft a Narrative**: Open a Discussion or Issue describing the industrial problem you want to solve.
2.  **Define the Nodes**: Create the Flow-Based YAML nodes that represent the logic.
3.  **Validate via Spec**: Update the documentation in `docs/specs/` to include Mermaid diagrams of the new intent.
4.  **Sovereign Review**: Ensure your contribution doesn't introduce cloud dependencies or proprietary DCS brand names.

## 🏗️ Synthesis Standards
- No source code in the main branch.
- Valid Mermaid diagrams for all control flows.
- Adherence to the **SAGE.md** safety calculus.

## 🛡️ Respecting Sovereignty
Contributors must respect the **Sovereign Shield**. Never push your local `tools/` implementation or environment secrets to the public repository.
